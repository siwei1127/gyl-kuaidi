# gyl-kuaidi

Project scaffold and baseline repository structure.

## Structure
- docs/   Documentation and design notes.
- src/    Application or library source code.
- tests/  Automated tests.

## Getting started
1. Choose a language/runtime and add tooling.
2. Implement core modules in src/.
3. Add tests under tests/.

## Notes
- Keep documentation in docs/.
- Keep configuration at the repo root.
